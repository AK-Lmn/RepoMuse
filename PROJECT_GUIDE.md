# RepoMuse - Project Documentation

## 1. App Overview
- **What this app does:** RepoMuse is an Android application that helps developers generate professional portfolio case studies from their GitHub repositories. It uses AI to automatically write project pitches, features, tech stack details, and resume bullets based on repository metadata and basic user inputs.
- **Main purpose of RepoMuse:** To streamline the process of documenting projects, making it easier for developers to showcase their work to recruiters and peers.
- **Main user flow:** 
  1. **Login:** Users authenticate using their GitHub account via Firebase (or skip for offline usage).
  2. **Dashboard (Home):** Users see a list of their saved case studies.
  3. **Add Project:** Users paste a GitHub URL and click "Generate with AI" to fill in details automatically.
  4. **Review/Edit:** Users can edit the generated case study, tags, and features.
  5. **Export:** Users can export the finished case study to PDF.

## 2. Tech Stack
- **Framework:** Android (Native) using Kotlin and **Jetpack Compose** for building the UI. (Note: This is not a React/Node.js app).
- **Styling/Library:** Material Design 3 (M3) components natively provided by Jetpack Compose.
- **Authentication:** Firebase Authentication with GitHub OAuth provider.
- **Database/Storage:** 
  - Local Storage: **Room Database** (SQLite wrapper for Android).
  - Cloud Storage: **Firebase Firestore** (used to sync projects across devices for logged-in users).
- **AI/API Integration:** 
  - **Gemini API** (Google) for generating case study text (points to stable `gemini-3.5-flash`).
  - **GitHub REST API** (Unauthenticated) for fetching repository metadata (description, primary language, topics).

## 3. Folder Structure
- `app/src/main/java/com/example/ui/screens/`
  - `HomeScreen.kt`: The main dashboard displaying the list of saved projects.
  - `LoginScreen.kt`: Handles GitHub OAuth login and offline mode.
  - `AddEditProjectScreen.kt`: The form to input a GitHub URL, call the AI generator, and save the project details. Also contains robust case-insensitive regex extraction blocks.
  - `ProjectDetailScreen.kt`: Shows the full case study and allows exporting to PDF.
- `app/src/main/java/com/example/ui/`
  - `ProjectViewModel.kt`: Manages the state of the application, coordinating inserts, updates, and UUID generations.
- `app/src/main/java/com/example/data/`
  - `Project.kt`: The data model (Room Entity) representing a single portfolio project using a `String` UUID primary key.
  - `ProjectDao.kt` & `AppDatabase.kt`: The local SQLite database configurations.
  - `ProjectRepository.kt`: Coordinates data between local Room DB and Firebase Firestore.
  - `GeminiService.kt`: Handles HTTP requests to the Gemini AI API (`gemini-3.5-flash`).
  - `GitHubService.kt`: Handles HTTP requests to the public GitHub API.
- `app/src/main/java/com/example/utils/`
  - `PdfExporter.kt`: Utility to generate a PDF file from the project data.

## 4. Authentication Flow
- **GitHub OAuth:** The app leverages Firebase Auth to handle the GitHub OAuth flow. When the user taps "Continue with GitHub", the app triggers `FirebaseAuth.getInstance().startActivityForSignInWithProvider()` using `github.com` as the provider.
- **Firebase Connection:** The `google-services.json` file configures the connection to the specific Firebase project.
- **Post Sign-in:** Upon successful login, the app navigates the user to the `HomeScreen`. The user's ID (`uid`) is now used to sync their projects to their personal Firestore collection.
- **Location:** The login logic is primarily located in `LoginScreen.kt`.

## 5. Database / Firebase Structure
- **Firestore Collections:** Data is stored under `users/{userId}/projects/{projectId}` where `projectId` is a globally unique UUID string generated by the client.
- **What is saved:** Each project document contains fields like `title`, `pitch`, `problem`, `githubUrl`, `liveUrl`, `features`, `techStack`, `resumeBullets`, `caseStudy`, and `tags`.
- **Security Rules Needed:** To ensure users only access their own data, Firestore rules should look like this:
  ```javascript
  match /users/{userId}/projects/{projectId} {
    allow read, write: if request.auth != null && request.auth.uid == userId;
  }
  ```

## 6. Main Features
- **Login:** Allows cloud syncing of data. Users can bypass this using "Continue Offline", storing data only locally.
- **Dashboard:** Shows a list of generated projects. Tapping a project opens its details. Tapping the FAB (+) starts a new project.
- **Add Project & GitHub Repo Input:** Provides form fields to enter project details manually or paste a GitHub URL.
- **AI Case Study Generation:** Takes the provided URL, fetches metadata (like topics and description), and sends a prompt to Gemini. The response is parsed into specific fields (Pitch, Problem, Features, Challenges, etc.).
- **Saved Projects:** Projects are saved locally in the Room DB and synced to Firestore (if logged in).
- **Export/Copy:** Projects can be exported as a standard PDF document to the device's Downloads folder for easy sharing.

## 7. Component Map (Jetpack Compose Functions)
Since this is an Android app using Jetpack Compose (not React), the UI is built with Composable functions:
- **`LoginScreen`**: Renders the GitHub login button and handles the Firebase Auth task.
- **`HomeScreen`**: Fetches the list of `Project` objects from the `ProjectViewModel` and displays them in a `LazyColumn` (list).
- **`AddEditProjectScreen`**: Contains `OutlinedTextField` inputs and an adaptive `SnackbarHost` to report fetch or generation errors. Depends on `GeminiService` and `GitHubService` when the "Generate with AI" button is clicked.
- **`ProjectDetailScreen`**: Displays the finalized data. Depends on `PdfExporter` when the user clicks the export button.

## 8. Environment Variables
- **`GEMINI_API_KEY`**: Found in the `.env` file. It is required to authenticate requests to the Google Gemini API.
- **`KEYSTORE_PATH`**, **`STORE_PASSWORD`**, **`KEY_PASSWORD`**: Optional variables used for signing the release APK during production builds.
*Note: The `.env` file should never be committed to source control to protect the `GEMINI_API_KEY`.*

## 9. How to Run Locally
As this is an Android Native App, standard Node.js commands (`npm install`, `npm run dev`) **do not apply**. Instead, use the Android build system:
1. **Prerequisites:** Install Android Studio.
2. **Open Project:** Open the root directory of the project in Android Studio.
3. **Environment Setup:** Create a `.env` file in the root directory and add your Gemini API key:
   `GEMINI_API_KEY=your_actual_key_here`
4. **Firebase Setup:** 
   - Go to the Firebase Console and create a project.
   - Add an Android app and register your application ID (`com.aistudio.repomuse.mxyzp`).
   - Enable GitHub Authentication in the Firebase Console (Requires setting up an OAuth App in GitHub).
   - Download `google-services.json` and place it in the `app/` folder.
5. **Run the App:** Click the green "Run" (Play) button in Android Studio, or run `./gradlew installDebug` from the terminal to install it on a connected device or emulator.

## 10. Current Limitations
- **Network Error Handling:** Network requests to Gemini and GitHub will fail if the device is completely offline, but are now safely caught and reported via user-friendly Snackbar notifications rather than dumping raw error messages into fields.
- **GitHub API Rate Limits:** The unauthenticated GitHub API call is subject to strict rate limits (60 requests per hour).
